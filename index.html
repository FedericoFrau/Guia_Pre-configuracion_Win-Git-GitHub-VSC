- Crear Token en GitHub, estando dentro de tu cuenta de GitHub:

-> Settings -> Developers Settings -> Personal access tokens -> Tokens (classic) -> Generate new token -> Generate new token (classic) -> (completar tu contraseña de GitHub):

  • Note: Dar un nombre abreviado para el token.
  • Expiration: seleccionar una duración de validez para el token.
  • Solo tildar el item ‘repo’ (se auto-seleccionan las opciones anidadas correspondientes).

-> Generate token -> (copiar token enlistado a continuación del tilde verde en un lugar seguro, no será posible verlo nuevamente)


- Crear credencial en Windows:

-> Panel de control -> Cuentas de usuario -> Administrador de credenciales -> Credenciales de Windows -> Agregar una credencial genérica ->:

  • Direccion de red o internet: git:https://github.com
  • Nombre de usuario: ‘tu_usuario_de_GitHub’
  • Contraseña: ‘token_generado_en_GitHub’

-> Guardar


- Configurar Token de GitHub con el espacio local:

- Abre una terminal en tu computadora.

- Verifica si tienes configurado nombre de usuario y correo electrónico global en Git ejecutando los comandos:

git config --global user.name

git config --global user.email

-Si los valores están vacíos, configúralos utilizando los comandos:

git config --global user.name "tu_usuario"

git config --global user.email "tu_correo@ejemplo.com"

- Copia el token generado en GitHub

- Ejecuta el siguiente comando para almacenar el token en la configuración de Git:

git config --global credential.https://github.com.username “tu_usuario”

Reemplaza "tu_usuario" con tu nombre de usuario en la plataforma Git.

- Ejecuta el siguiente comando para configurar el token:

config --global credential.helper store

Esto hará que Git almacene las credenciales, incluido el token, en el disco de manera persistente.

- Verifica si has almacenado un token en la configuración de Git ejecutando el comando:

git config --global credential.https://github.com.username

Si muestra tu nombre de usuario significa que has almacenado un token. Git utilizará el token para autenticarte cuando realices operaciones que lo requieran (clonar, push, pull).
